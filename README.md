# AI-for-finance-Hack-2025

# Financial RAG & Consensus System
Данный проект представляет собой систему автоматизированного ответа на вопросы (Q&A) на основе предоставленной базы знаний. Проект разработан специально для задач, требующих точных ответов на русском языке с использованием контекста (например, для финтех-хакатонов).

# Основные возможности
Быстрый поиск контекста: Векторизация текстов на основе пересечения множеств слов (Keyword-based retrieval), что обеспечивает высокую скорость работы без использования тяжелых эмбеддинг-моделей.

Контроль языка: Встроенная проверка is_russian и автоматический перевод ответов, если модель перешла на английский.

Очистка от «галлюцинаций» и вежливости: Система автоматически удаляет фразы-филлеры типа "согласно базе данных..." или "я не нашел ответа, но...", делая ответ более естественным.

Взвешенное голосование: Оценка качества ответов на основе их длины, структуры (списки) и наличия примеров.

Отказоустойчивость: Обработка ошибок API и последовательная генерация с сохранением прогресса через tqdm.

# Технологический стек
Язык: Python 3.9+

Библиотеки: pandas, openai, tqdm, python-dotenv, numpy.

LLM: Gemma 3 27B (через OpenRouter или кастомный API-шлюз).

# Требования к данным
Для работы скрипта в корневой директории должны находиться следующие файлы:

train_data.csv — база знаний. Должна содержать колонку text.

questions.csv — список вопросов. Должна содержать колонку Вопрос.

.env — файл с ключами API:

Фрагмент кода
OPENAI_API_KEY=your_key_here
EMBEDDER_API_KEY=your_key_here
 Логика работы (Pipeline)
Preprocessing: Скрипт загружает базу знаний и создает легковесные "векторы" (множества слов) для каждой записи.

Retrieval: Для каждого вопроса ищется топ-3 наиболее релевантных фрагмента текста из базы знаний через пересечение ключевых слов.

Generation:

Формируется системный промпт, запрещающий упоминать источники информации.

Запрос отправляется в LLM.

Post-processing:

Удаление лишних фраз (Regex-очистка).

Проверка на русский язык (если порог кириллицы ниже 60% — запуск принудительного перевода).

Quality Analysis: Ответ анализируется на "структурность" (наличие списков, примеров и объема), что влияет на итоговый вес в голосовании.

Saving: Результаты сохраняются в submission_final.csv.

# Заметки разработчику
Эвристика поиска: Метод find_relevant_context_fast эффективен для поиска по точным терминам, но не учитывает синонимы. Если точность поиска будет низкой, рекомендуется добавить эмбеддинги.

Лимиты API: В коде настроен base_url для специфичного хакатон-шлюза. При использовании официального OpenAI API измените base_url в функциях генерации и перевода.
